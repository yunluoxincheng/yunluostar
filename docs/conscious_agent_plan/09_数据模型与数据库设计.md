# 09. 数据模型与数据库设计

## 1. 推荐数据库组合

MVP：

- SQLite：结构化状态与日志。
- sqlite-vec：向量检索，优先服务 Bot Runtime 的本地/self-hosted 运行形态。

进阶：

- PostgreSQL：长期结构化数据。
- LanceDB / Chroma / Qdrant / Milvus：向量数据库。
- Neo4j：知识图谱和因果关系。

## 2. 核心表

### episodes

情节记忆。

```sql
CREATE TABLE episodes (
  id TEXT PRIMARY KEY,
  timestamp TEXT,
  session_id TEXT,
  context TEXT,
  user_intent TEXT,
  agent_action TEXT,
  outcome TEXT,
  lesson TEXT,
  importance REAL,
  confidence REAL,
  status TEXT
);
```

### semantic_memories

语义记忆。

```sql
CREATE TABLE semantic_memories (
  id TEXT PRIMARY KEY,
  concept TEXT,
  content TEXT,
  confidence REAL,
  source_episode_id TEXT,
  last_updated TEXT,
  status TEXT
);
```

### self_model

自我模型。

```sql
CREATE TABLE self_model (
  id TEXT PRIMARY KEY,
  trait TEXT,
  value TEXT,
  confidence REAL,
  evidence TEXT,
  last_updated TEXT,
  mutable INTEGER
);
```

### user_model

用户模型。

```sql
CREATE TABLE user_model (
  id TEXT PRIMARY KEY,
  trait TEXT,
  value TEXT,
  confidence REAL,
  evidence TEXT,
  last_updated TEXT
);
```

### goals

目标系统。

```sql
CREATE TABLE goals (
  id TEXT PRIMARY KEY,
  type TEXT,
  description TEXT,
  priority REAL,
  status TEXT,
  mutable INTEGER,
  requires_user_approval INTEGER,
  created_at TEXT,
  updated_at TEXT
);
```

### working_memory_snapshots

工作记忆快照。

```sql
CREATE TABLE working_memory_snapshots (
  id TEXT PRIMARY KEY,
  session_id TEXT,
  timestamp TEXT,
  state_json TEXT
);
```

### reflections

反思记录。

```sql
CREATE TABLE reflections (
  id TEXT PRIMARY KEY,
  episode_id TEXT,
  what_worked TEXT,
  what_failed TEXT,
  lessons TEXT,
  self_update_candidate TEXT,
  created_at TEXT
);
```

### conflicts（后续候选表）

冲突记录。当前实现没有独立 `conflicts` 表；目标冲突 MVP 使用 `goals.conflict_of` 和 `audit_logs.action = "conflict"` 记录冲突关系与审计信息。下面表结构保留为后续显式冲突模型的候选设计。

```sql
CREATE TABLE conflicts (
  id TEXT PRIMARY KEY,
  memory_a_id TEXT,
  memory_b_id TEXT,
  conflict_type TEXT,
  description TEXT,
  resolution_status TEXT,
  created_at TEXT
);
```

## 3. 向量索引

建议为以下对象生成 embedding：

- semantic_memories.content
- self_model.trait + value
- user_model.trait + value
- reflections.lessons（后续可选）
- episodes.context + event + lesson（后续可选；当前实现优先为 semantic/user/self 状态生成向量）

## 4. 审计日志

所有重要更新都应写入 audit_logs：

```sql
CREATE TABLE audit_logs (
  id TEXT PRIMARY KEY,
  timestamp TEXT,
  actor TEXT,
  action_type TEXT,
  target_table TEXT,
  target_id TEXT,
  before_value TEXT,
  after_value TEXT,
  reason TEXT
);
```
