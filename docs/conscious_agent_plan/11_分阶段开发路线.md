# 11. 分阶段开发路线

## 阶段 0：理论与需求定义

目标：明确项目边界。

任务：

- 写清楚不宣称主观意识。
- 定义主体型 Agent 的功能指标。
- 确定第一个应用场景：研究型 Agent。

交付物：

- 项目 README
- 架构设计文档
- 风险边界文档

## 阶段 1：Cognitive RAG

目标：让 Agent 有结构化长期记忆。

任务：

- 实现 episodes 表。
- 实现 semantic_memories 表。
- 实现 user_model 表。
- 实现 memory_awakener。
- 实现 memory_consolidator。

验收：

- 能记录一次对话为情节记忆。
- 能从多次对话抽取稳定语义记忆。
- 能在下一轮对话中主动使用过去记忆。

## 阶段 2：Working Memory

目标：让 Agent 有当前状态。

任务：

- 实现 WorkingMemory 类。
- 支持当前目标、当前假设、当前风险。
- 支持会话快照。

验收：

- Agent 能解释自己当前正在处理什么。
- Agent 能在多步骤任务中保留状态。

## 阶段 3：Self Model

目标：让 Agent 有动态自我模型。

任务：

- 实现 self_model 表。
- 实现 self_model_updater。
- 建立能力、限制、行为倾向数据结构。

验收：

- 用户反馈能改变 Agent 的回答策略。
- Agent 能说出自己的能力边界和限制。

## 阶段 4：Goal System

目标：让 Agent 能维护长期目标。

任务：

- 实现 goals 表。
- 实现 goal_manager。
- 实现目标优先级排序。
- 实现目标冲突检测。

验收：

- Agent 能区分短期问题和长期研究目标。
- Agent 能推荐下一步研究任务。

## 阶段 5：Metacognition

目标：让 Agent 能检查自己的认知过程。

任务：

- 实现 confidence_check。
- 实现 risk_check。
- 实现 reflection。
- 实现 conflict detection。

验收：

- Agent 能识别不确定问题。
- Agent 能在回答后生成反思。
- Agent 能发现前后记忆冲突。

## 阶段 6：Planning & Action

目标：让 Agent 能进行多步骤行动。

任务：

- 实现 planner。
- 实现 action_executor。
- 实现执行日志。
- 实现失败恢复。

验收：

- Agent 能拆解一个研究任务。
- Agent 能执行工具调用。
- Agent 能根据结果调整计划。

## 阶段 7：World Model

目标：让 Agent 能预测行动后果。

任务：

- 实现简单因果规则库。
- 实现行动后果预测。
- 实现风险预测。

验收：

- Agent 能说明为什么选择某个行动。
- Agent 能预测不同方案的优缺点。

## 阶段 8：持续运行闭环

目标：整合所有模块。

任务：

- 实现定期反思。
- 实现记忆巩固任务。
- 实现长期目标维护。
- 实现状态恢复。

验收：

- Agent 多天运行后保持连续性。
- Agent 能从过去经验中改变未来行为。
