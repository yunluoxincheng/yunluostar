# 11. 分阶段开发路线

## 当前实现状态（2026-05-21）

- 阶段 0 已完成：项目边界、架构设计、风险原则和 CLI-first 技术路线已形成文档。
- 阶段 1 已完成 MVP：结构化长期记忆、记忆唤醒、反思、巩固、纠错、审计日志和 CLI 检查命令已实现。
- 阶段 2 已完成 MVP：WorkingMemory 值对象、快照表、恢复/注入/保存流水线和 `/wm` 只读查看已实现。
- 向量检索已提前落地：sqlite-vec、embedding client、embedding store 和复合评分已接入记忆唤醒。
- 下一步建议进入阶段 4 的 Goal System MVP；阶段 3 的动态 Self Model 已有基础更新能力，但仍缺少核心自我边界种子和更清晰的自我能力模型。

## 阶段 0：理论与需求定义

目标：明确项目边界。

任务：

- 写清楚不宣称主观意识。
- 定义主体型 Agent 的功能指标。
- 确定第一个应用场景：CLI-first 研究型 Agent。

交付物：

- 项目 README
- 架构设计文档
- 风险边界文档
- TypeScript + CLI-first MVP 技术路线

## 阶段 1：Cognitive RAG

状态：MVP 已完成。

目标：让 CLI Agent 有结构化长期记忆。

任务：

- 建立 TypeScript / Node.js 项目骨架。
- 实现 CLI chat 命令。
- 实现 episodes 表。
- 实现 semantic_memories 表。
- 实现 user_model 表。
- 实现 memory_awakener。
- 实现 memory_consolidator。

验收：

- 能通过命令行启动一轮对话。
- 能记录一次对话为情节记忆。
- 能从多次对话抽取稳定语义记忆。
- 能在下一轮对话中主动使用过去记忆。

## 阶段 2：Working Memory

状态：MVP 已完成。

目标：让 Agent 有当前状态。

任务：

- 实现 WorkingMemory 类。
- 支持当前目标、当前假设、当前风险。
- 支持会话快照。

验收：

- Agent 能解释自己当前正在处理什么。
- Agent 能在多步骤任务中保留状态。

## 阶段 3：Self Model

状态：部分完成。当前已有 `self_model` 表、更新候选、证据记录和查看接口；仍需补充初始化核心自我模型、能力/限制结构化边界和更明确的 self_model_updater 策略。

目标：让 Agent 有动态自我模型。

任务：

- 实现 self_model 表。
- 实现 self_model_updater。
- 建立能力、限制、行为倾向数据结构。

验收：

- 用户反馈能改变 Agent 的回答策略。
- Agent 能说出自己的能力边界和限制。

## 阶段 4：Goal System

状态：下一阶段建议实施。当前已有 `goals` 表和只读查看接口，但尚未实现 goal manager、目标分层、优先级排序、建议目标、审批流和冲突检测。

目标：让 Agent 能维护长期目标。

任务：

- 实现 goals 表。
- 实现 goal_manager。
- 实现目标优先级排序。
- 实现目标冲突检测。

验收：

- Agent 能区分短期问题和长期研究目标。
- Agent 能推荐下一步研究任务。

## 阶段 5：Metacognition

目标：让 Agent 能检查自己的认知过程。

任务：

- 实现 confidence_check。
- 实现 risk_check。
- 实现 reflection。
- 实现 conflict detection。

验收：

- Agent 能识别不确定问题。
- Agent 能在回答后生成反思。
- Agent 能发现前后记忆冲突。

## 阶段 6：Planning & Action

目标：让 Agent 能进行多步骤行动。

任务：

- 实现 planner。
- 实现 action_executor。
- 实现执行日志。
- 实现失败恢复。

验收：

- Agent 能拆解一个研究任务。
- Agent 能执行工具调用。
- Agent 能根据结果调整计划。

## 阶段 7：World Model

目标：让 Agent 能预测行动后果。

任务：

- 实现简单因果规则库。
- 实现行动后果预测。
- 实现风险预测。

验收：

- Agent 能说明为什么选择某个行动。
- Agent 能预测不同方案的优缺点。

## 阶段 8：持续运行闭环

目标：整合所有模块。

任务：

- 实现定期反思。
- 实现记忆巩固任务。
- 实现长期目标维护。
- 实现状态恢复。

验收：

- Agent 多天运行后保持连续性。
- Agent 能从过去经验中改变未来行为。

## 产品形态路线

当前 MVP 采用 CLI-first：

```text
CLI → Agent Runtime → SQLite / Vector Index → LLM / Tools
```

CLI 是第一入口，用于验证主体闭环、调试内部状态和运行评估实验。

后续可在 Agent Runtime 稳定后增加：

- Hono / Fastify API：为 Web 或外部系统提供服务接口。
- Web Research Console：可视化记忆、目标、自我模型和反思。
- Desktop App：作为长期本地运行和个人工作台入口。
