# 11. 分阶段开发路线

> **方向转型说明（2026-05-22）：** yunluostar 已从 CLI-first agent 研究原型转型为多平台认知型 Agentic Bot 平台。认知核心的已完成阶段保持不变，新增 Bot 平台专属阶段。详见 `docs/goals/archive/pivot-to-cognitive-bot-platform.md`。

> **后续实现路线（2026-06-01）：** Bot Runtime MVP 完成后的执行路线见 `docs/plans/2026-06-01-bot-platform-implementation-roadmap.md`。当前 active OpenSpec change 是 `stabilize-bot-platform-mvp`，用于稳定显式 adapter contract、demo surface、cognitive trace、plugin safety 和 scope isolation。

## 当前实现状态（2026-06-01）

### 已完成的认知核心能力

- 阶段 0 已完成：项目边界、架构设计、风险原则和技术路线已形成文档。
- 阶段 1 已完成 MVP：结构化长期记忆、记忆唤醒、反思、巩固、纠错、审计日志和 CLI 检查命令已实现。
- 阶段 2 已完成 MVP：WorkingMemory 值对象、快照表、恢复/注入/保存流水线和 `/wm` 只读查看已实现。
- 阶段 4 已完成 MVP：Goal System — 分层目标、建议目标审批、冲突检测、目标管理命令。
- Runtime-backed 架构边界已完成：CLI / Runtime / Protocol / Runtime-Client 边界清晰。
- 向量检索已提前落地：sqlite-vec、embedding client、embedding store 和复合评分已接入记忆唤醒。

### 当前阶段：Bot Platform MVP 稳定化

Bot 平台方向转型和第一轮 Runtime MVP 已完成。按 `docs/goals/archive/pivot-to-cognitive-bot-platform.md`，以下基础能力已经落地：

| Bot 平台阶段 | 目标 | 状态 |
|------|------|------|
| Phase 0 | 方向锁定 — 文档统一为 Bot 平台方向 | ✅ 已完成 |
| Phase 1 | Bot Protocol & Scope — 平台无关协议 + scope 映射 | ✅ 已完成 |
| Phase 2 | Runtime Bot Message Path — Bot 消息接入认知流水线 | ✅ 已完成 |
| Phase 3 | HTTP webhook / generic Bot API — 非 CLI Bot 入口 | ✅ 已完成基础入口 |
| Phase 4 | WebChat-Compatible API — POST /v1/bot/message | ✅ 已完成 |
| Phase 5 | Plugin System MVP — manifest, registry, hooks | ✅ 已完成 |

当前 active change `stabilize-bot-platform-mvp` 不重做上述能力，而是把它们稳定成更清晰的 Bot Platform MVP：

- 显式 adapter contract 和 `src/adapters/generic-http/`。
- dedicated generic HTTP adapter route。
- `scripts/demo-bot-platform.ts` 作为最小非 CLI demo surface。
- Bot response trace metadata 稳定化。
- 插件 capability、timeout、错误隔离和 trace 补强。
- Bot scope isolation 测试强化。

### 下一步方向（先稳定 MVP，再扩展生态）

- Adapter Contract 与 generic HTTP adapter。
- 非 CLI demo script；WebChat 页面可选，不做完整 dashboard。
- Cognitive Trace 和行为级认知验收。
- Plugin Safety MVP。
- 生产 webhook/auth 设计文档和局部实现。
- 之后再考虑 IM 平台适配器、Web Dashboard、插件市场、Docker/Kubernetes 部署。

### 待完善的认知能力（与 Bot 平台并行推进）

- 阶段 3 的动态 Self Model 已有基础更新能力，但仍缺少核心自我边界种子和更清晰的自我能力模型。
- 阶段 5 Metacognition：认知过程的监控和调整。
- 阶段 6 Planning & Action：多步骤计划、执行日志、失败恢复。
- 阶段 7 World Model：行动后果预测与风险推演。
- 阶段 8 持续运行闭环：长期定期反思、目标维护和跨天连续性。

## 阶段 1：Cognitive RAG

状态：MVP 已完成。

目标：让 Agent 有结构化长期记忆；当前长期记忆主要服务 Bot Runtime，CLI chat 默认不写入长期认知状态。

任务：

- 建立 TypeScript / Node.js 项目骨架。
- 实现早期 CLI chat 命令；当前 CLI chat 仅作为 ephemeral debug path。
- 实现 episodes 表。
- 实现 semantic_memories 表。
- 实现 user_model 表。
- 实现 memory_awakener。
- 实现 memory_consolidator。

验收：

- 能通过 Bot API 启动一轮认知对话；CLI 仅用于 admin/debug。
- 能记录一次对话为情节记忆。
- 能从多次对话抽取稳定语义记忆。
- 能在下一轮对话中主动使用过去记忆。

## 产品形态路线

yunluostar 是多平台认知型 Bot 平台：

```text
Bot 适配器 (WebChat / HTTP Webhook / IM) → Bot Runtime → Cognitive Pipeline → SQLite / Vector Index → LLM / Tools
```

CLI 保留为 admin/debug/developer 工具，不是主要产品入口。

后续可在 Bot Runtime 稳定后增加：

- IM 平台适配器：QQ、Telegram、Discord、Slack、飞书、钉钉、企业微信
- Web Dashboard：可视化记忆、目标、自我模型、反思、插件和适配器管理
- 插件市场
- 独立 coding agent 产品（与 Bot 平台记忆系统分离）
