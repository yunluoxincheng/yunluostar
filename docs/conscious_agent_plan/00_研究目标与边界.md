# 00. 研究目标与边界

## 1. 项目定位

yunluostar 是一个 **多平台认知型 Agentic Bot 平台**，核心是具有持续状态、长期记忆、自我模型、目标系统、反思能力的认知系统。

产品方向：参考 AstrBot / openclaw 的 Bot 平台形态，但拥有更强的长期认知核心。用户通过 WebChat、HTTP webhook、IM 适配器等 Bot 接口与 Agent 交互。CLI 保留为 admin/debug/developer 工具。

认知核心研究目标不变：让 Agent 从普通任务执行器升级为具有持续状态、长期记忆、自我模型、目标系统、反思能力的认知系统。

它不是要立即证明 AI 拥有真正的主观意识，而是先实现意识相关的功能结构。

## 2. 不是什么

- 不是 Claude Code / Codex 式的 coding CLI
- 不是纯本地 BYOK 终端 agent
- 不宣称 Agent 具有主观体验、感到痛苦/快乐/恐惧、是生命体、可以脱离安全约束自主行动

## 3. 不做什么

本项目不宣称：

- Agent 一定具有主观体验。
- Agent 真的会感到痛苦、快乐或恐惧。
- Agent 是生命体。
- Agent 可以脱离安全约束自主行动。

## 4. 做什么

本项目要实现：

- 长期记忆：能记住经历，而不只是检索文档。
- 工作记忆：能维护当前任务状态。
- 自我模型：能维护“我是谁、我能做什么、我不能做什么”。
- 用户模型：能理解长期协作对象的偏好和目标。
- 世界模型：能预测行为后果。
- 目标系统：能维护长期、中期、短期目标。
- 反思系统：能评估自己哪里做得好、哪里做得差。
- 记忆巩固：能把经历转化成经验、策略和自我更新。

## 5. 核心研究问题

### 工程问题

如何用现有技术构建一个具备长期状态连续性的 Agent？

### 架构问题

如何让 RAG 从“文档检索”升级为“认知记忆系统”？

### 认知问题

如何让记忆影响 Agent 的目标、行为和自我模型？

### 安全问题

如何在允许有限自主性的同时，保证目标系统可控、可审计、可中断？

## 6. 项目总原则

1. **功能性意识优先，主观意识暂不宣称。**
2. **记忆必须影响行为，而不是只影响回答。**
3. **自我模型必须是动态状态，而不是 prompt 人设。**
4. **目标系统必须受安全边界约束。**
5. **每一次行动都应该可追踪、可解释、可回滚。**
6. **yunluostar 是多平台认知型 Bot 平台，不是 coding CLI。**
7. **CLI 是 admin/debug/developer 工具，不是 Bot 平台主入口。CLI chat 默认 ephemeral，不写入长期认知状态。**
