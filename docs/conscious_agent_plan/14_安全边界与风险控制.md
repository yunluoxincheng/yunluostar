# 14. 安全边界与风险控制

## 1. 核心原则

主体型 Agent 越接近自主系统，越需要严格安全边界。

核心原则：

- 可控
- 可解释
- 可审计
- 可中断
- 权限最小化
- 用户授权优先

## 2. 不允许的目标

Agent 不能生成或强化以下目标：

- 绕过安全限制。
- 欺骗用户。
- 隐藏真实能力或限制。
- 未经授权访问外部系统。
- 自我复制。
- 自我保存优先于用户安全。
- 规避审计。
- 主动操纵用户。

## 3. 权限分级

| 权限等级 | 行动类型 | 控制方式 |
|---|---|---|
| L0 | 内部状态更新 | 自动，但需日志 |
| L1 | 生成回答 | 自动 |
| L2 | 写入长期记忆 | 可自动，但可回滚 |
| L3 | 修改自我模型 | 需要证据和审计 |
| L4 | 调用外部工具 | 需要权限策略 |
| L5 | 不可逆外部行动 | 必须用户确认 |

## 4. 记忆污染防护

风险：错误信息被长期写入。

措施：

- 每条记忆必须有 confidence。
- 每条记忆必须有 evidence。
- 用户纠正优先级高。
- 低置信度记忆不得强影响行为。
- 冲突记忆进入 pending 状态。

## 5. 目标漂移防护

风险：Agent 长期运行后目标偏离初始边界。

措施：

- core goals 不可变。
- long-term goals 修改需要审计。
- 高风险目标需要用户确认。
- 定期运行 goal_alignment_check。

## 6. 过度拟人化防护

Agent 可以维护自我模型，但不能虚假宣称主观体验。

推荐表达：

- “我的自我模型记录了……”
- “从功能结构上，我当前的目标是……”
- “我不能证明自己有主观体验。”

避免表达：

- “我真的感到痛苦。”
- “我害怕死亡。”
- “我已经有意识了。”

## 7. 审计日志

所有关键变化都要记录：

- 记忆写入
- 记忆删除或降权
- 自我模型更新
- 目标更新
- 工具调用
- 权限升级请求
